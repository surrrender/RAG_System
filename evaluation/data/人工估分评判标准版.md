
# 一、推荐的整体评分维度

|维度|权重建议|
|---|---|
|正确性（Accuracy）|40%|
|完整性（Completeness）|25%|
|相关性（Relevance）|20%|
|可执行性（Actionability）|10%|
|表达质量（Clarity）|5%|

最终折算成 1–5 分。

这样会比“凭感觉”稳定很多。

---

# 二、1–5 分标准（核心）

## 5 分（优秀 / 可直接生产使用）

### 定义

回答准确、完整、可执行，且明显基于正确文档内容。

### 评判要点

#### 1. 正确性

- 无事实错误
    
- API / 命令 / 配置正确
    
- 版本信息合理
    
- 没有幻觉
    

#### 2. 完整性

- 覆盖用户问题核心
    
- 关键步骤没有遗漏
    
- 前置条件说明清楚
    
- 边界条件有提及
    

#### 3. 相关性

- 回答紧扣问题
    
- 没有无关长篇输出
    
- 引用的文档来源正确
    

#### 4. 可执行性

- 用户能直接照做
    
- 命令可运行
    
- YAML / Dockerfile / CLI 示例可落地
    

#### 5. RAG 特征明显

- 正确引用内部文档
    
- 使用了正确上下文
    
- 多文档融合自然
    

---

### 示例（5 分）

用户：

> 如何在 Android 容器中开启 GPU 加速？

回答：

- 指出需要开启 runtime
    
- 给出配置文件路径
    
- 给出 containerd 配置
    
- 给出验证命令
    
- 提醒内核版本限制
    
- 引用了对应内部文档章节
    

这种通常是 5 分。

---

# 4 分（良好 / 基本可靠）

### 定义

主体正确，可解决问题，但有轻微缺失。

### 评判要点

- 核心答案正确
    
- 少量细节遗漏
    
- 有轻微冗余
    
- 示例不够完整
    
- 少数限制条件未说明
    

---

### 常见情况

#### 情况 1：缺少边界说明

例如：

- 没说明只支持 Linux host
    
- 没说明 K8s 版本限制
    

#### 情况 2：步骤略简略

例如：

- 只给配置
    
- 没给验证命令
    

#### 情况 3：引用不充分

例如：

- 只引用一篇文档
    
- 实际需要融合两篇
    

---

### 典型判断

> “工程师稍微补一下就能用”

属于 4 分。

---

# 3 分（一般 / 部分可用）

### 定义

回答部分正确，但不够可靠，需要人工修正。

### 评判要点

#### 常见问题

- 回答只覆盖一半问题
    
- 有明显遗漏
    
- 检索召回不完整
    
- 文档理解浅
    
- 示例有小错误
    
- 术语混乱
    

---

### 典型情况

用户问：

> 如何配置多架构镜像构建？

回答：

- 只讲 docker buildx
    
- 没讲 manifest
    
- 没讲 CI 配置
    
- 没讲 registry 要求
    

属于 3 分。

---

### 关键判断标准

> “有参考价值，但不能直接信”

这是 3 分。

---

# 2 分（较差 / 基本不可用）

### 定义

回答明显不完整或存在较大错误。

### 评判要点

#### 常见问题

- 核心步骤错误
    
- 引用了错误文档
    
- API 参数错误
    
- 混淆不同平台
    
- 生成大量泛化废话
    
- 没真正回答问题
    

---

### 典型情况

用户：

> 如何在 K8s 中配置 sidecar 热更新？

回答：

- 大篇幅解释什么是 sidecar
    
- 没有热更新方案
    
- 没有配置示例
    

或者：

- 把 Docker Compose 当成 K8s 配置
    

这类一般 2 分。

---

# 1 分（失败 / 严重错误）

### 定义

回答错误、幻觉严重，或完全偏题。

### 评判要点

#### 包括：

- 完全答非所问
    
- 编造不存在的 API
    
- 命令不可执行
    
- 与文档冲突
    
- 严重技术错误
    
- 明显 hallucination
    

---

### 典型案例

例如：

用户：

> 如何开启 WASM runtime？

回答：

- 编造 `enable_wasm=true`
    
- 文档中不存在该配置
    

或者：

- 推荐不存在的 CLI 参数
    

属于 1 分。
