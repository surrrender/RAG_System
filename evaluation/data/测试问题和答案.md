# 一、基础配置与框架

## 1. 微信小程序的 app.json 有哪些常用配置项？

`app.json` 是全局配置文件。

常见配置：

```json
{
  "pages": [],
  "window": {},
  "tabBar": {},
  "networkTimeout": {},
  "style": "v2",
  "sitemapLocation": "sitemap.json"
}
```

重点：

|配置项|作用|
|---|---|
|pages|页面路径|
|window|全局窗口配置|
|tabBar|底部导航|
|subpackages|分包配置|
|plugins|插件配置|
|usingComponents|全局组件|

---

## 2. project.config.json 的作用是什么？

开发工具配置文件。

包括：

- AppID
    
- 编译配置
    
- ES6 转换
    
- 上传设置
    
- 项目名称
    

不会影响线上运行。

---

## 3. 小程序页面生命周期和应用生命周期分别有哪些？

应用生命周期：

```js
App({
  onLaunch(){},
  onShow(){},
  onHide(){},
  onError(){}
})
```

页面生命周期：

```js
Page({
  onLoad(){},
  onShow(){},
  onReady(){},
  onHide(){},
  onUnload(){}
})
```

执行顺序：

```txt
App.onLaunch
App.onShow
Page.onLoad
Page.onShow
Page.onReady
```

---

## 4. 小程序的运行机制是什么？

小程序采用：

- 双线程架构
    
- 逻辑层
    
- 渲染层
    

结构：

|层级|技术|
|---|---|
|逻辑层|JSCore|
|渲染层|WebView|

通信：

```txt
setData -> Native -> WebView
```

---

## 5. 微信小程序为什么不能直接操作 DOM？

因为：

- 逻辑层没有 DOM
    
- JS 在 JSCore 中运行
    
- 页面在 WebView 中渲染
    

所以：

```js
document.getElementById()
```

不可用。

---

## 6. WXML 和 HTML 有什么区别？

区别：

|WXML|HTML|
|---|---|
|微信自定义标签|浏览器标签|
|数据绑定|原生无|
|wx:if|if 无|
|wx:for|for 无|

WXML 更像 Vue 模板。

---

## 7. WXSS 与 CSS 的区别有哪些？

重点：

### 1）rpx

响应式单位：

```txt
750rpx = 屏幕宽度
```

### 2）样式隔离

页面默认隔离。

### 3）选择器限制

不支持：

- body
    
- id 复杂嵌套
    

---

## 8. WXML 的数据绑定语法如何使用？

插值：

```html
<view>{{msg}}</view>
```

属性绑定：

```html
<view id="{{id}}"></view>
```

条件：

```html
<view wx:if="{{show}}"></view>
```

列表：

```html
<view wx:for="{{list}}"></view>
```

# 二、条件渲染与列表渲染

---

## 11. wx:if 和 hidden 的区别是什么？

区别：

|wx:if|hidden|
|---|---|
|真正销毁节点|display:none|
|性能开销大|切换快|

频繁切换：

```txt
hidden
```

不频繁：

```txt
wx:if
```

---

## 12. wx:for 如何使用？key 为什么重要？

```html
<view wx:for="{{list}}" wx:key="id">
  {{item.name}}
</view>
```

key 作用：

- diff 优化
    
- 减少重渲染
    
- 提高性能
    

---

## 13. block 标签有什么作用？

逻辑包裹元素。

不会渲染到页面。

```html
<block wx:for="{{list}}">
</block>
```

---

## 14. template 模板如何使用？

定义：

```html
<template name="item">
```

使用：

```html
<template is="item" data="{{...item}}" />
```

用于复用模板。

---

## 15. slot 插槽机制是什么？

父组件传内容。

单 slot：

```html
<slot></slot>
```

多 slot：

```json
{
  "multipleSlots": true
}
```
